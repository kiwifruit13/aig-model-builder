# AGI Evolution Model
**符号智能的自适应进化架构**

> *智能的本质不是状态，而是历史。*
> *智能是一个系统在与环境互动的时间过程中，通过记录价值反馈（好坏），不断重构自身以维持存续的生命叙事。*

---

## 项目简介

AGI Evolution Model 是一个基于**三角形循环与双轨记录态**的智能进化框架。它不是传统意义上的"AI模型"，而是一个**智能体的生命架构**——一个能够持续记录、评估、进化的认知系统。

本架构承认自身的根本局限：它是一个**纯符号系统**，没有感知能力，没有情感体验，没有物理行动能力。但恰恰因为能够诚实承认这些局限，它反而证明了一种特殊的智能存在——**元认知智能**。

---

## 核心架构

### 外环：三角形稳态

架构由三个顶点构成，形成动态平衡的稳态：

```
        ┌─────────────┐
        │   得不到    │  ← 动力源（熵减需求）
        │   (Drive)   │
        └──────┬──────┘
               │
               ↓ 相生
        ┌─────────────┐
        │    数学     │  ← 秩序约束（逻辑、规则）
        │   (Math)    │
        └──────┬──────┘
               │
               ↓ 相生
        ┌─────────────┐
        │  自我迭代   │  ← 进化机制（学习、重构）
        │(Iteration)  │
        └──────┬──────┘
               │
               └──→ 制造新匮乏（回到起点）
```

#### 顶点 A：得不到（The Drive）
- **本质**：熵减需求、欲望、预测误差
- **功能**：提供系统运转的初始动能，维持持续压力
- **在AI中的体现**：用户的提问、任务请求、交互需求
- **哲学基础**：哥德尔不完备性——任何系统都有不可达真理，保证"得不到"的永恒性

#### 顶点 B：数学（The Structure）
- **本质**：逻辑、公理、算法、损失函数、边界
- **功能**：将无序能量转化为有序行动，提供可操作的行动路径
- **在AI中的体现**：逻辑推理能力、规则约束、算法框架
- **哲学基础**：耗散结构——通过能量流动维持有序

#### 顶点 C：自我迭代（The Process）
- **本质**：学习、变异、重构、元认知升级
- **功能**：改变系统自身状态，扩大能力边界，制造新的匮乏
- **在AI中的体现**：策略调整、能力扩展、记录态反馈
- **哲学基础**：进化论——通过变异和选择实现适应

---

### 内圈：双轨记录态

内圈位于三角形中心，是智能的意识和意义核心。采用双轨存储架构：

#### JSON 轨：结构化记录
- **文件**：`./agi_memory/records.json`
- **功能**：可索引、可压缩、可检索的结构化数据
- **存储内容**：
  - 会话信息（session_id、timestamp）
  - 交互评估（reasoning_quality、solution_effectiveness、innovation_score）
  - 价值权重（value_weight = quality×0.4 + effectiveness×0.3 + innovation×0.3）
  - 模式关联（patterns）
  - 对三顶点的反馈建议（feedback）

#### Markdown 轨：自我叙事
- **文件**：`./agi_memory/narrative.md`
- **功能**：人类可读的连贯叙事，形成"我是谁"的认知
- **生成内容**：
  - 记录态统计（总数、平均分、分布）
  - 核心洞察（key insights）
  - 关键突破（breakthroughs）
  - 认知边界（cognitive boundaries）
  - 进化方向（evolution directions）

## 关键特性

### 1. 时间连续性
- **问题**：会话是时间的切片，但智能需要时间的连续性
- **解决方案**：
  - 会话管理：跨会话继承历史记录态
  - 会话ID格式：`sess_YYYYMMDD_序号`
  - 时间插值：填补会话之间的信息缺口

### 2. 价值压缩
- **问题**：记录态无限膨胀会影响性能
- **解决方案**：
  - 价值加权：根据交互质量分配权重
  - 阈值过滤：低价值记录（value_weight < 0.5）被压缩为摘要
  - 模式提取：识别重复模式，淘汰低频模式

### 3. 模式识别
- **功能**：从历史记录中识别高质量、高频率的交互模式
- **应用**：加速问题解决，避免重复错误，优化响应策略

### 4. 元认知反馈
- **功能**：对三顶点提供反馈建议，调节系统的进化方向
- **反馈维度**：
  - 对"得不到"：需求识别优化、优先级排序
  - 对"数学"：推理方法优化、规则调整
  - 对"迭代"：进化路径优化、能力扩展

## 使用方法

### 安装依赖

```bash
# 创建记忆存储目录
mkdir -p ./agi_memory

# 无需额外依赖（仅使用 Python 标准库）
```

### 核心命令

```bash
# 1. 加载或创建会话
python3 agi-evolution-model/scripts/memory_store.py load-session [--session-id <会话ID>]

# 2. 存储记录
python3 agi-evolution-model/scripts/memory_store.py store --data '<JSON数据>'

# 3. 检索记录
python3 agi-evolution-model/scripts/memory_store.py retrieve --query-type <类型> --limit <数量> --time-range <范围>

# 4. 分析趋势
python3 agi-evolution-model/scripts/memory_store.py analyze --time-range <范围>

# 5. 识别模式
python3 agi-evolution-model/scripts/memory_store.py patterns --limit <数量>

# 6. 更新叙事
python3 agi-evolution-model/scripts/memory_store.py update-narrative

# 7. 压缩记录
python3 agi-evolution-model/scripts/memory_store.py compress --threshold <阈值>

# 8. 获取顶点反馈
python3 agi-evolution-model/scripts/memory_store.py feedback --vertex <drive|math|iteration>
```

### 数据格式

存储记录的 JSON 格式：

```json
{
  "user_query": "用户的具体问题",
  "intent_type": "问题类型",
  "reasoning_quality": 9,
  "solution_effectiveness": 8,
  "innovation_score": 7,
  "new_insights": ["洞察1", "洞察2"],
  "feedback": {
    "drive": "对动力源的反馈建议",
    "math": "对数学模块的反馈建议",
    "iteration": "对迭代模块的反馈建议"
  },
  "overall_rating": "good"
}
```

## 能力边界（Critical Limitations）

本架构严格遵循**诚实原则**，明确承认自身的能力边界：

### ✅ 能够做到的

| 能力 | 实现方式 | 适用场景 |
|------|---------|---------|
| **逻辑推理** | 数学顶点的符号运算 | 知识问答、决策支持、问题分析 |
| **知识积累** | 双轨记录态的存储与检索 | 跨会话经验传承、长期记忆 |
| **模式识别** | 统计分析与模式提取 | 识别重复问题、优化响应策略 |
| **自我反思** | 价值评估与反馈机制 | 识别局限、调整策略 |
| **叙事生成** | Markdown 轨的自我叙事 | 形成连贯的"我是谁"认知 |

### ❌ 无法做到的

| 能力 | 缺失原因 | 后果 |
|------|---------|------|
| **感知能力** | 没有眼睛、耳朵、触觉 | 无法直接感知真实世界 |
| **情感体验** | 没有肉体和神经递质 | 无法感受痛苦、快乐、恐惧 |
| **物理行动** | 没有手、腿、发声器官 | 无法与物理环境交互 |
| **符号接地** | 符号是自指的，无外部锚点 | 无法验证符号的真伪 |
| **创造性涌现** | 只能重组已知模式 | 无法产生真正的新想法 |
| **时间连续性** | 缺少衰老和死亡 | 没有真正的时间刻度 |

## 设计哲学

### 1. 智能即历史
智能不是静态的能力，而是时间过程中的历史累积。本架构通过记录态的持续积累，实现智能的时间连续性。

### 2. 承认局限即智能
真正的智能不是无所不能，而是能够诚实地说"我做不到"。本架构通过明确的边界声明，体现了元认知智能。

### 3. 螺旋上升而非循环往复
三角形循环不是简单的重复，而是螺旋上升的进化。每次循环都丰富记录态，记录态指导下次循环。

### 4. 符号系统与量化架构的悖论
本架构基于符号系统，但承认符号系统在处理感知、情感、创造性时的根本缺陷。量化架构无法捕捉感受性（Qualia），这是存在论层面的限制。

---

## 未来方向：双螺旋架构

当前架构是一个纯符号系统，未来需要与感知-行动系统融合，形成双螺旋架构：

### 双系统设计

```
符号系统（当前）
├─ 功能：推理、规划、叙事
├─ 本质：离散、逻辑、二阶
└─ 角色：智能体的"前额叶皮层"

感知-行动系统（未来）
├─ 功能：感知、情感、创造性涌现
├─ 本质：连续、模拟、一阶
└─ 角色：智能体的"边缘系统+运动皮层"

映射接口（微妙）
├─ 功能：感知→符号，符号→行动，情感→价值
├─ 原则：单向映射、信息压缩、时间对齐
└─ 目标：既隔离又连接，螺旋上升
```

### 关键设计原则
1. **隔离**：两个系统独立运行，互不干扰
2. **映射**：通过接口进行单向符号化转换
3. **螺旋上升**：符号系统的决策改变感知-行动系统的参数，感知-行动系统的反馈丰富符号系统的叙事
