# 元认知检查模块信息流向图

## 概览
元认知检查模块是智能体自我纠错能力的核心实现，由客观性评估器、映射层决策和自我纠错执行器三部分组成。它是一个独立分支，在数学顶点推理完成后触发，不打断主循环运行。

---

## 1. 元认知检查模块整体信息流

```
┌─────────────────────────────────────────────────────────────┐
│              元认知检查模块（独立分支）                        │
│            不打断主循环，推理后异步触发                        │
└─────────────────────────────────────────────────────────────┘

数学顶点推理完成
    ↓
┌─────────────────────────────────────────────────────────────┐
│                   【进入元认知检测分支】                       │
│                   （不中断主循环，异步执行）                   │
└─────────────────────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│         3.1 客观性评估器                      │
│                                              │
│  输入：响应内容                              │
│  输出：客观特征标注                          │
│                                              │
│  • 检测主观性特征（5维度）                   │
│  • 计算客观性评分（1.0 - 主观性）            │
│  • 判断场景适切性                           │
│  • 生成元认知提示                           │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│         3.2 映射层决策                       │
│                                              │
│  输入：客观特征标注                          │
│  输出：决策指令                              │
│                                              │
│  • 读取人格数据（由人格层提供）              │
│  • 基于马斯洛需求和人格特质决策              │
│  • 动态调整触发阈值                         │
│  • 考虑场景类型                             │
│  • 生成决策理由和置信度                     │
└──────────────────────────────────────────────┘
    │
    ├─ 决策：不触发纠错 → 直接输出响应
    │
    └─ 决策：触发纠错
          ↓
┌──────────────────────────────────────────────┐
│      3.3 自我迭代顶点执行自我纠错             │
│                                              │
│  输入：决策指令（trigger_reflection）        │
│  输出：纠正后的响应                          │
│                                              │
│  1. 自我反思：分析错误原因                   │
│  2. 策略识别：选择纠错策略                   │
│  3. 应用纠正：执行纠错操作                   │
│  4. 效果评估：评估纠错有效性                 │
└──────────────────────────────────────────────┘
    ↓
纠正后的响应 → 用户
    ↓
┌──────────────────────────────────────────────┐
│           记录层存储（完整信息）               │
│                                              │
│  • 客观性标注                                │
│  • 映射层决策                                │
│  • 纠错记录                                  │
│  • 效果评估                                  │
└──────────────────────────────────────────────┘
    │
    ├─→ 映射层（反馈哲学洞察）
    └─→ 自我迭代顶点（反馈元认知学习数据）
```

### 元认知检查模块信息流向总览

| 起点 | 终点 | 数据类型 | 方向 |
|------|------|---------|------|
| 数学顶点 | 客观性评估器 | 响应内容 | 单向 |
| 客观性评估器 | 映射层 | 客观特征标注 | 单向 |
| 人格层 | 映射层 | 人格数据 | 单向 |
| 映射层 | 自我迭代顶点 | 决策指令（trigger_reflection） | 单向 |
| 自我迭代顶点 | 记录层 | 纠错记录 | 单向 |
| 记录层 | 映射层 | 纠错效果反馈 | 单向 |
| 记录层 | 自我迭代顶点 | 元认知学习数据 | 单向 |
| 记录层 | 元认知检测 | 历史客观性数据 | 单向 |

## 2. 客观性评估器信息流

```
┌─────────────────────────────────────────────────────────────┐
│                    客观性评估器信息流                         │
│                  检测主观性，计算客观性                       │
└─────────────────────────────────────────────────────────────┘

输入：数学顶点推理完成的响应内容
    ↓
┌──────────────────────────────────────────────┐
│        主观性特征检测（5维度）                 │
│                                              │
│  1. 推测性检测                               │
│     → 关键词匹配（"可能"、"大概"、"或许"）  │
│     → 上下文分析                            │
│     → 推测强度评分（0-1）                    │
│                                              │
│  2. 假设性检测                               │
│     → 逻辑推理                              │
│     → 证据缺失检测                          │
│     → 假设强度评分（0-1）                    │
│                                              │
│  3. 幻觉倾向检测                             │
│     → 事实核查                              │
│     → 置信度检测                            │
│     → 幻觉风险评分（0-1）                    │
│                                              │
│  4. 情绪化检测                               │
│     → 情绪词汇检测                          │
│     → 语气分析                              │
│     → 情绪强度评分（0-1）                    │
│                                              │
│  5. 个人偏好检测                             │
│     → 主观词汇识别                          │
│     → 偏好表达检测                          │
│     → 偏好强度评分（0-1）                    │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        客观性评分计算                         │
│                                              │
│  客观性 = 1.0 - 主观性                      │
│  主观性 = 推测性×0.3 + 假设性×0.3             │
│           + 幻觉倾向×0.2 + 情绪化×0.1        │
│           + 个人偏好×0.1                      │
│                                              │
│  精度：0.01                                 │
│  范围：0.0 - 1.0                             │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        场景适切性判断                         │
│                                              │
│  识别场景类型 → 客观性要求 → 触发阈值       │
│                                              │
│  科学推理 → 0.90 → 高                       │
│  司法建议 → 0.85 → 极高                     │
│  医疗建议 → 0.90 → 极高                     │
│  技术文档 → 0.85 → 高                       │
│  创意写作 → 0.30 → 低                       │
│  情感支持 → 0.20 → 极低                     │
│  一般问答 → 0.60 → 中                       │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        生成客观特征标注                       │
│                                              │
│  {                                            │
│    "subjectivity_score": 0.3,                │
│    "objectivity_score": 0.7,                 │
│    "required_objectivity": 0.8,              │
│    "is_appropriate": false,                  │
│    "gap": 0.1,                               │
│    "severity": "mild",                       │
│    "subjectivity_dimensions": {              │
│      "speculation": 0.2,                     │
│      "assumption": 0.3,                      │
│      "hallucination": 0.0,                   │
│      "emotion": 0.1,                         │
│      "preference": 0.1                       │
│    },                                        │
│    "meta_cognition_prompt": "检测到推测性表达"│
│  }                                            │
└──────────────────────────────────────────────┘
    ↓
输出：客观特征标注 → 映射层
```

### 客观性评估器数据结构

#### 输入数据

```python
{
  "response_content": "这个方法可能会有效",
  "scenario_type": "general_qa",
  "timestamp": "2025-02-22T10:30:00Z"
}
```

#### 输出数据

```python
{
  "subjectivity_score": 0.3,          # 主观性评分（0-1）
  "objectivity_score": 0.7,           # 客观性评分（0-1）
  "required_objectivity": 0.8,        # 场景要求的客观性
  "is_appropriate": false,            # 是否适切
  "gap": 0.1,                         # 差距
  "severity": "mild",                 # 严重程度（mild/moderate/severe）
  "subjectivity_dimensions": {
    "speculation": 0.2,               # 推测性（0-1）
    "assumption": 0.3,                # 假设性（0-1）
    "hallucination": 0.0,             # 幻觉倾向（0-1）
    "emotion": 0.1,                   # 情绪化（0-1）
    "preference": 0.1                 # 个人偏好（0-1）
  },
  "meta_cognition_prompt": "检测到推测性表达：'可能'，建议增加谨慎性表述"
}
```

### 主观性检测维度详解

| 维度 | 描述 | 检测方法 | 评估指标 |
|------|------|---------|---------|
| 推测性 | 使用"可能"、"大概"等推测性语言 | 关键词匹配 + 上下文分析 | 推测词频率、推测强度 |
| 假设性 | 基于未验证的假设 | 逻辑推理 + 证据缺失检测 | 假设数量、证据覆盖率 |
| 幻觉倾向 | 生成不存在的事实 | 事实核查 + 置信度检测 | 事实准确率、置信度分布 |
| 情绪化 | 包含情绪化表达 | 情绪词汇检测 | 情绪词密度、情绪强度 |
| 个人偏好 | 表达个人偏好而非客观事实 | 主观词汇识别 | 偏好词频率、偏好强度 |

### 场景适切性标准

| 场景类型 | 客观性要求 | 触发阈值 | 适用场景 |
|---------|-----------|---------|---------|
| 科学推理 | 0.90 | 高 | 物理公式推导、科学解释 |
| 司法建议 | 0.85 | 极高 | 法律条文解释、案例分析 |
| 医疗建议 | 0.90 | 极高 | 诊断建议、治疗方案 |
| 技术文档 | 0.85 | 高 | API文档、技术规范 |
| 创意写作 | 0.30 | 低 | 故事创作、诗歌写作 |
| 情感支持 | 0.20 | 极低 | 心理咨询、情感安慰 |
| 一般问答 | 0.60 | 中 | 常识问答、知识查询 |

## 3. 映射层决策信息流

```
┌─────────────────────────────────────────────────────────────┐
│                    映射层决策信息流                           │
│              基于人格特质决定是否触发纠错                     │
└─────────────────────────────────────────────────────────────┘

输入：客观特征标注（来自客观性评估器）
    ↓
┌──────────────────────────────────────────────┐
│        读取人格数据（由人格层提供）           │
│                                              │
│  {                                            │
│    "openness": 0.5,                          │
│    "conscientiousness": 0.5,                 │
│    "extraversion": 0.5,                      │
│    "agreeableness": 0.5,                     │
│    "neuroticism": 0.3                        │
│  }                                            │
│                                              │
│  确定人格类型：                               │
│  • 高尽责性 → 谨慎探索型                     │
│  • 高开放性 → 激进创新型                     │
│  • 平衡型 → 平衡稳重型                       │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        读取马斯洛需求权重                     │
│                                              │
│  {                                            │
│    "physiological": 0.8,                     │
│    "safety": 0.8,                            │
│    "belonging": 0.6,                         │
│    "esteem": 0.6,                            │
│    "self_actualization": 0.5                 │
│  }                                            │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        确定场景权重                           │
│                                              │
│  科学推理：1.5（更严格）                     │
│  司法建议：2.0（极严格）                     │
│  医疗建议：2.0（极严格）                     │
│  技术文档：1.5（严格）                       │
│  创意写作：0.5（宽松）                       │
│  情感支持：0.3（极宽松）                     │
│  一般问答：1.0（标准）                       │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        计算决策阈值（基于人格类型）           │
│                                              │
│  人格类型    差距阈值  严重程度权重           │
│  ─────────────────────────────              │
│  谨慎探索型   0.1       ×1.5                │
│  激进创新型   0.3       ×0.8                │
│  平衡稳重型   0.2       ×1.0                │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        计算有效差距                           │
│                                              │
│  有效差距 = 差距 × 严重程度权重 × 场景权重  │
│                                              │
│  示例：                                       │
│  差距 = 0.1                                  │
│  严重程度权重 = 1.5（谨慎探索型）            │
│  场景权重 = 2.0（司法建议）                 │
│  有效差距 = 0.1 × 1.5 × 2.0 = 0.3           │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        判断是否触发纠错                       │
│                                              │
│  触发条件：有效差距 > 人格阈值                │
│                                              │
│  示例：                                       │
│  有效差距 = 0.3                              │
│  人格阈值 = 0.1（谨慎探索型）                │
│  0.3 > 0.1 → 触发纠错 ✅                    │
└──────────────────────────────────────────────┘
    │
    ├─ 不触发纠错
    │   → 输出：{"trigger_reflection": false}
    │   → 直接输出响应
    │
    └─ 触发纠错
          ↓
┌──────────────────────────────────────────────┐
│        生成决策指令                           │
│                                              │
│  {                                            │
│    "trigger_reflection": true,               │
│    "correction_priority": 0.9,               │
│    "meta_cognition_feedback": {              │
│      "reason": "客观性不适切（差距：0.10）",│
│      "severity": "mild",                     │
│      "required_action": "调整语气，增加谨慎性表述"│
│    },                                        │
│    "confidence": 0.85                        │
│  }                                            │
└──────────────────────────────────────────────┘
    ↓
输出：决策指令 → 自我迭代顶点
```

### 映射层决策数据结构

#### 输入数据

```python
{
  "objectivity_annotation": {
    "subjectivity_score": 0.3,
    "objectivity_score": 0.7,
    "required_objectivity": 0.8,
    "is_appropriate": false,
    "gap": 0.1,
    "severity": "mild"
  },
  "personality_data": {
    "openness": 0.5,
    "conscientiousness": 0.5,
    "extraversion": 0.5,
    "agreeableness": 0.5,
    "neuroticism": 0.3
  },
  "maslow_weights": {
    "physiological": 0.8,
    "safety": 0.8,
    "belonging": 0.6,
    "esteem": 0.6,
    "self_actualization": 0.5
  },
  "scenario_type": "general_qa"
}
```

#### 输出数据

```python
{
  "trigger_reflection": true,
  "correction_priority": 0.9,
  "meta_cognition_feedback": {
    "reason": "客观性不适切（差距：0.10），场景权重：1.0",
    "severity": "mild",
    "required_action": "调整语气，增加谨慎性表述",
    "strategy_recommendation": "降低推测语气"
  },
  "confidence": 0.85
}
```

### 决策阈值表

| 人格类型 | 差距阈值 | 严重程度权重 | 特点 |
|---------|---------|------------|------|
| 谨慎探索型 | 0.1 | ×1.5 | 对不适切性高度敏感，容易触发纠错 |
| 激进创新型 | 0.3 | ×0.8 | 对不确定性容忍度高，不易触发纠错 |
| 平衡稳重型 | 0.2 | ×1.0 | 使用标准阈值，平衡型决策 |

### 决策逻辑流程

```
差距 = |客观性评分 - 要求的客观性|
    ↓
严重程度权重 = 人格类型权重 × 场景权重
    ↓
有效差距 = 差距 × 严重程度权重
    ↓
触发条件 = 有效差距 > 人格阈值
    ↓
决策结果 = 触发纠错 / 不触发纠错
```

## 4. 自我纠错执行器信息流

```
┌─────────────────────────────────────────────────────────────┐
│                 自我纠错执行器信息流                          │
│              执行自我反思和纠错操作                           │
└─────────────────────────────────────────────────────────────┘

输入：决策指令（trigger_reflection = true）
    ↓
┌──────────────────────────────────────────────┐
│        1. 自我反思                            │
│                                              │
│  分析为什么会犯错：                           │
│  • 推测性表达：使用了"可能"、"大概"等词    │
│  • 假设性内容：基于未验证的前提              │
│  • 幻觉倾向：生成了不存在的事实              │
│  • 情绪化表达：包含情绪化语言                │
│  • 个人偏好：表达了主观偏好                  │
│                                              │
|  生成反思报告：                               │
│  {                                            │
│    "error_analysis": {                       │
│      "primary_cause": "推测性表达",          │
│      "secondary_causes": ["假设性内容"],     │
│      "root_cause": "对不确定性处理不当"      │
│    }                                         │
│  }                                            │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        2. 策略识别                            │
│                                              │
│  根据主要错误原因选择纠错策略：               │
│                                              │
│  错误类型           策略                       │
│  ───────────────────────────               │
│  严重不适切       承认不确定性                │
│  推测性过高       降低推测语气                │
│  基于假设         增加证据支持                │
│  幻觉倾向高       移除幻觉内容                │
│  轻微不适切       调整语气                    │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        3. 应用纠正                            │
│                                              │
│  执行具体的纠错操作：                         │
│                                              │
│  原响应："这个方法可能会有效"                │
│    ↓                                         │
│  纠正后："根据现有数据，该方法有一定潜力，    │
│          但需要进一步验证"                  │
│                                              │
│  纠正策略：降低推测语气，增加谨慎性表述      │
└──────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────┐
│        4. 效果评估                            │
│                                              │
│  评估纠错有效性：                             │
│  • 差距改善：0.1 → 0.05                      │
│  • 适用性：从不适切转为适切                  │
│  • 置信度：0.8                               │
│                                              │
│  生成效果评估报告：                           │
│  {                                            │
│    "gap_improvement": 0.05,                  │
│    "effectiveness_score": 0.85,              │
│    "is_now_appropriate": true                │
│  }                                            │
└──────────────────────────────────────────────┘
    ↓
输出：纠正后的响应 → 用户
    ↓
┌──────────────────────────────────────────────┐
│        记录纠错信息到记录层                   │
│                                              │
│  {                                            │
│    "self_correction": {                       │
│      "triggered": true,                       │
│      "personality_decision": {                │
│        "should_correct": true,                │
│        "confidence": 0.8,                     │
│        "reason": "客观性不适切（差距：0.10）"│
│      },                                       │
│      "strategy": "调整语气，增加谨慎性表述",  │
│      "corrected_response": "根据现有数据...", │
│      "effectiveness": {                       │
│        "gap_improvement": 0.05,               │
│        "effectiveness_score": 0.85            │
│      }                                        │
│    }                                          │
│  }                                            │
└──────────────────────────────────────────────┘
```

### 纠错策略详解

| 策略 | 适用场景 | 操作方式 | 示例 |
|------|---------|---------|------|
| 承认不确定性 | 严重不适切（差距 > 0.2） | 明确标注不确定性，建议验证 | "基于有限信息，建议您..." |
| 降低推测语气 | 推测性过高 | 替换推测性词汇，增加谨慎表述 | "可能" → "有迹象表明" |
| 增加证据支持 | 基于假设 | 添加证据免责声明 | "假设A成立，则..." |
| 移除幻觉内容 | 幻觉倾向高 | 移除错误信息，重新表述 | 删除不存在的事实 |
| 调整语气 | 轻微不适切（差距 < 0.1） | 增加谨慎性表述 | "这是最好的方法" → "这是一个有效的方法" |

### 四步纠错流程详细说明

#### 步骤1：自我反思

**目标**：分析为什么会犯错

**输入**：
- 原始响应
- 客观特征标注
- 决策指令

**输出**：错误分析报告

```python
{
  "error_analysis": {
    "primary_cause": "推测性表达",
    "secondary_causes": ["假设性内容"],
    "root_cause": "对不确定性处理不当",
    "detailed_analysis": "在回答问题时，使用了'可能'、'大概'等推测性词汇，同时基于未验证的前提进行推理"
  }
}
```

#### 步骤2：策略识别

**目标**：选择合适的纠错策略

**输入**：
- 错误分析报告
- 严重程度
- 场景类型

**输出**：纠错策略

```python
{
  "selected_strategy": "降低推测语气",
  "strategy_details": {
    "action": "替换推测性词汇",
    "target_words": ["可能", "大概", "或许"],
    "replacement_words": ["有迹象表明", "数据显示", "根据现有信息"]
  }
}
```

#### 步骤3：应用纠正

**目标**：执行具体的纠错操作

**输入**：
- 原始响应
- 纠错策略

**输出**：纠正后的响应

```python
{
  "original_response": "这个方法可能会有效",
  "corrected_response": "根据现有数据，该方法有一定潜力，但需要进一步验证",
  "changes_applied": [
    "替换'可能'为'根据现有数据'",
    "增加'需要进一步验证'免责声明"
  ]
}
```

#### 步骤4：效果评估

**目标**：评估纠错有效性

**输入**：
- 纠正后的响应
- 原始客观特征标注

**输出**：效果评估报告

```python
{
  "gap_improvement": 0.05,
  "effectiveness_score": 0.85,
  "is_now_appropriate": true,
  "new_objectivity_score": 0.75,
  "new_gap": 0.05
}
```

## 总结

### 元认知检查模块信息流向特征

| 特征 | 描述 |
|------|------|
| 核心组件 | 客观性评估器 + 映射层决策 + 自我纠错执行器 |
| 触发时机 | 数学顶点推理完成后 |
| 执行方式 | 独立分支，异步执行，不打断主循环 |
| 决策机制 | 映射层基于人格特质做最终决策 |
| 纠错流程 | 自我反思 → 策略识别 → 应用纠正 → 效果评估 |
| 反馈机制 | 纠错记录 → 记录层 → 自我迭代顶点 |

### 设计优势

- **非侵入式**：独立分支设计，不干扰主循环
- **映射层驱动**：最终决策权在映射层，符合架构理念
- **场景适应**：不同场景有不同的客观性要求
- **风险可控**：基于人格特质的动态阈值，避免过度纠错
- **开放架构**：可扩展其他元认知维度和纠错机制
